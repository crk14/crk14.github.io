两个全局变量

  1. __dirname 当前文件所在的目录的绝对路径
  2. __filename 当前文件的绝对路径

NodeJS 内置的 path 模块

  1. path.resolve(…path) 推荐使用
  2. path.join(…path)

PS: 这两种的区别

  1. join() 只是简单的对多个路径参数做合并。而 resolve() 中，如果有某个 根路径的参数，这个根路径参数前面的参数将会被忽略掉。

  2. resolve() 总是会返回绝对路径。而join()就得到参数是怎样。

express 中 request

Express 帮我们在 request 对象上，做了一些封装，新增了如下一些常用属性or方法

  1. req.query：获取URL的查询参数串 (用于get)
  2. req.body: 获取请求主体（post数据，需要设置中间件函数 json与urlencode）
  3. req.cookies：获取Cookies (需要使用 cookie-parser 中间件)
  4. req.hostname / req.ip：获取主机名和IP地址
  5. req.params：获取路由的parameters
  6. req.path：获取请求路径
  7. req.protocol：获取协议类型
  8. req.get()：获取指定的HTTP请求头

express 中 response

Express 帮我们在 response 对象上，做了一些封装，新增了如下一些常用属性or方法

  1. res.set()：设置HTTP头，传入object可以一次设置多个头
  2. res.status()：设置HTTP状态码
  3. res.send()：传送HTTP响应
  4. res.redirect()：设置响应的Location HTTP头，并且设置状态码302
  5. res.json()：传送JSON响应
  6. res.cookie(name，value [，option])：设置Cookie
  7. res.clearCookie()：清除参数Cookie
  8. res.download()：传送指定路径的文件
  9. res.sendFile(path [，options] [，fn])：传送指定路径的文件 -会自动根据文件extension设定Content-Type
  10. res.render(view [, locals] [, callback]) 渲染一个view

中间件

中间件的概念，在一起请求与响应的周期中。它一般都是在请求与响应的周期中对request对象或response做修改。

一个中间件其实就是一个函数，这个函数接收到3个参数，分别是 req、res、next。next()是让整个的流程往下面继续执行的。如果你不调用的话，这次的请求响应就直接断开了。

请求 -> 中间件1 -> 中间件2 - 中间件3 -> 响应

  2. express.urlencode()
  3. cookieParser()
