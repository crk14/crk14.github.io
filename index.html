计算属性能实现的效果，用方法或者监听都能实现。

  1. 跟方法比较，会节省性能，因为计算属性有缓存。
  2. 跟监听比较，更方便，代码书写更顺畅。

v-if 和 v-show 有什么区别

  1. v-if 才是真正的条件渲染，条件为ture时渲染，条件为false时销毁。而 v-show 只是简单的控制元素的css样式来做到显示隐藏

  2. v-if 有更大的切换开销，v-show 有更大的初始渲染开销。所以一般需要做频繁的切换的元素就可以使用 v-show 来控制。其余就可以使用 v-if

  3. v-if 有配套的 v-else v-else-if 。 而 v-show 只是孤零零的一个。

  4. v-if 可以配合 template 去使用。而 v-show 不能。

vue路由总结

vue-router 是 vue 官方提供的一套 路由插件，能够帮我们实现单页面应用。
单页面应用
什么是单页面应用（Single Page Application | SPA）
整个项目只有一个 html 文件的，然后页面的跳转不会刷新页面的，就是单页面应用
什么是多页面应用
项目中有多个 html 文件的，页面之间的跳转是通过 a 标签或者 location.href 的方式，
页面跳转会刷新浏览器，就是多页面应用
为什么现在会使用单页面应用呢
为了使 h5 网站像原生 app 贴近，比如实现页面的转场动画。这时就出现了单页面应用。

webpack

现代化前端最流行的一款 构建打包工具
前端构建打包工具的发展历史
grunt
gulp
fis
webpack
…
webpack 打包构建原理
基于某个入口文件，分析这个文件中的依赖。然后继续往下寻找依赖的依赖。循环往复，最后没有依赖了，把找到的全部都打包到一起。
webpack 核心
entry (入口)
output (出口)
loaders (转换器) - webpack 将依赖都看成是一个一个的模块。但是默认情况只能识别 js与json 两种格式的模块。如果想要打包图片、样式等资源的话，就需要相应的loader转换器来进行转换操作，转换之后webpack才能识别。
plugins (插件) - 有一些操作转换器也实现不了，这时就可以交给插件来实现。比如说要将某个文件直接复制到出口里面等操作。
mode (打包模式 4.x 版本之后提供的) - 开发 | 生产
